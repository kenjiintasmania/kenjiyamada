## 際立つ特徴（新作に効かせる）

1. **リスニング比重が高い**（大問1がA〜Dと厚い）。しかもリスニング内に**英作文(3語以上)・1語補充**が混ざる＝「聞く→書く」の複合。
2. **岡山ローカル題材**が頻出：後楽園(Korakuen)、岡山駅、岡山の地元料理。→ 新作でも「県内の地名・行事・名所」を題材化すると傾向に寄せられる（内容は新規創作）。
3. **記述・条件英作文が各所に分散**：4語以上(日記)、6語抜き出し(話し合い)、2語抜き出し＋日本語記述(長文)。自動採点だけでは測れない設問が多い＝自己採点(`en`/`jp`)や別途AI添削と相性。
4. **図表(ちらし)読み取り**：日付・料金・場所・条件の突合。→ 「定員を実数で明記」など、唯一解になる設計を守る（QA①で頻発した不具合の予防）。
5. **語彙・文法の小問**：語形変化(現在完了の過去分詞など)、共通語、並べかえ、感情語(glad/sorry)×補語の組合せ。中3配当の文法（受動態・現在完了・関係代名詞・間接疑問）が軸。

## 配点の見立て（一般的傾向・要確認）
- 100点満点。リスニング 30点前後、読解・記述が中心。設問ごとの配点は年度で変動。
- 既存模試は各設問に `pt` を持ち合計100点。新作も**満点採点が通る**よう `npm run check` を必須に。

## 新作生成の方針（reference-only）
- 上の**形式・難易度・配点バランスだけ**を踏襲し、**本文・設問・選択肢はすべて新規創作**。
- 題材は岡山の地名・名所・行事を**創作的に**使う（実在の固有名詞の事実誤りを避け、必要なら架空の町名 Wakaba/Midori 等を使う＝既存模試の流儀）。
- 出力先 `mogi/data/*.js`（既存スキーマ）→ `npm run check` 100点 → `[gen:okayama]` PR。
- ⚠️ 著作権：過去問の文・図の**そのままの再現は禁止**。傾向の要約とオリジナル化のみ。
